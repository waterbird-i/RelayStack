# RelayStack Intro Voiceover

## 中文解说稿

先看一个真实问题。
AI 已经能把很多代码写完。
可是，工作一换人，往往就断了。

上一轮做到哪？为什么这么改？哪里有风险？下一步怎么验？
这些答案如果散在聊天和 diff 里，接手的人就得从头翻一遍。

RelayStack 解决的就是这件事。
它不追求让 agent 写更多代码，而是让已经做过的工作，可以被交接。
它会把当前现场整理成一份 handoff snapshot，也就是交接快照。

这份快照会说清楚七件事：
当前目标是什么，已经完成了什么，改了哪些文件，为什么这么推进。
还有风险在哪里，下一步做什么，以及怎么判断真的完成。

如果是多个 agent 并行，RelayStack 还会把边界摆出来。
谁负责探索，谁负责实现，谁负责审查。
每个 agent 写了哪些范围，哪些结论被采纳，有没有冲突，验证结果是什么，都会留下证据。

所以它的工作流设计很克制。
第一版不是平台，也没有账号、数据库或者任务看板。
它就是一组仓库内的 skills，加上一份 Markdown 快照，直接嵌入现有的 AI coding 流程。

生成快照时，它读取本地证据：
Git 状态、diff、最近提交、项目文档，以及可选的 AgentRecord。
有就引用，没有就写明未发现，不靠猜。

最后，RelayStack 的效果也不是用“总结得好不好看”来衡量。
我们看 handoff success rate：
新的 owner 只读这份快照，能不能答对交接问题，能不能在几分钟内继续下一步。

这就是 RelayStack 的亮点。
它把 AI 工作从一次性输出，变成可交接、可验证、可继续的团队资产。

## 使用方式

本视频使用本机中文 TTS 生成 `public/voiceover.wav`，并在 Remotion 合成时接入音轨。
TTS 文本在 `public/voiceover.txt`，其中包含显式停顿控制。
