# Backlog Attractor

This directory holds prioritized work that the team may actually pick up.

Keep:

- candidate features
- known issues worth fixing
- status and owner when known
- the smallest useful next action
- verification notes after closure
- roadmap slices from `rs-roadmap`
- final feature and issue status from `rs-feat-accept` and `rs-issue-fix`

Do not keep:

- full implementation checklists
- abandoned brainstorming trails
- private agent task records

Typical writers: `rs-roadmap`, `rs-brainstorm` after an idea is stable enough,
`rs-feat-accept`, `rs-issue-fix`.
