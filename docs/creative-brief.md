# RelayStack 创意简介

## 1. 创意概述

RelayStack 是一个以 skill 形式落地的 AI 编程工作流增强层。

它的核心创意是：当一段 AI 编程工作需要被人或另一个 agent 接手时，不再依赖散落在聊天记录、Git diff 和临时说明里的上下文，而是生成一份可继续执行的 handoff snapshot。

这份快照不是普通总结，而是一份交接证据：它说明当前目标、改动范围、已完成事项、风险、阻塞、下一步和验证方式，让接手者可以快速判断“现在做到哪了，接下来该怎么做”。

## 2. 解决的问题

AI 编程已经能完成越来越多具体任务，但团队协作中的断点仍然明显：

- 上下文分散在聊天、代码 diff、任务描述和个人判断里
- 多个 agent 并行时，职责边界和写入范围不清晰
- 换人或换 agent 后，接手者不知道为什么这样改
- 需求变化后，很难判断哪些决策和文件受影响
- Demo 能跑起来，但缺少明确的验证路径和继续行动依据

RelayStack 解决的不是“让 agent 写更多代码”，而是让 agent 的工作能够被交接、被验证、被继续。

## 3. 目标用户

- 使用 AI 编程工具的开发者
- 需要多人接力完成任务的研发团队
- 同时调度多个 agent 的项目负责人
- 需要展示 AI 交付过程的 hackathon 或原型团队
- 希望降低上下文切换成本的技术负责人

## 4. 使用场景

### 人接手 agent 的工作

开发者让 agent 完成一部分任务后，触发 RelayStack 生成 handoff snapshot。接手者只阅读快照，就能了解当前状态、风险和下一步动作。

### agent 接手 agent 的工作

上一个 agent 结束执行后留下任务记录。下一个 agent 读取 handoff snapshot，继续完成未结束的工作，而不是重新理解整段聊天。

### 多 agent 并行后的合流

多个 agent 分别负责探索、实现、审查或验证。RelayStack 汇总它们的任务记录，展示职责、写入范围、采纳状态、冲突和验证结论。

### 演示和复盘 AI 交付过程

项目团队可以用 handoff snapshot 展示 AI 工作不是一次性黑盒输出，而是包含证据、边界和验收路径的可交付过程。

## 5. 核心功能

### 交接快照生成

读取当前工作区状态和用户提供的交接字段，生成 Markdown 格式的 handoff snapshot。

快照包含：

- 当前目标和阶段
- Git 状态、diff 摘要、改动文件和最近提交
- 已完成和未完成事项
- 阻塞、风险和需要确认的问题
- 项目上下文
- 下一步动作、验证方式和完成标志

### Agent 任务记录接入

支持读取本地 JSON 或 Markdown 形式的 AgentRecord，把不同 agent 的任务、输入、输出、结论、写入范围和验证结果纳入交接快照。

### 并行边界呈现

当多个 agent 参与同一轮工作时，快照会呈现：

- 每个 agent 的职责
- 写入范围
- 任务状态
- 采纳状态
- 显式冲突
- 验证结果
- 潜在文件范围重叠

### 交接成功率验证

RelayStack 用一组交接问题衡量快照是否真正可用：

```text
handoff success rate = correctly answered handoff questions / total handoff questions
```

接手者能回答的问题越多，说明交接质量越高。

## 6. 作品亮点

- **skill-first**：第一版不做平台和复杂服务，直接嵌入现有 AI 编程工作流
- **证据优先**：基于真实工作区状态生成，不停留在聊天总结
- **交接导向**：关注“下一个人或 agent 能不能继续做”，而不是代码行数
- **并行可见**：把多 agent 的职责、写入范围和冲突显式呈现
- **低成本落地**：输出 Markdown 文件，不依赖数据库、账号系统或远端服务
- **可验证**：用交接问题和 handoff success rate 判断实际效果

## 7. 预期价值

### 对开发者

减少重新理解上下文的时间，降低接手 AI 生成代码时的不确定性。

### 对团队

让 AI 编程过程更可追踪，减少“这个改动为什么存在”的沟通成本。

### 对多 agent 工作流

让多个 agent 的工作可以被汇总、审查和继续，而不是变成互相割裂的输出。

### 对演示和评审

把 AI 工作从“看起来完成了”提升到“能证明可交接、可验证、可继续”。

## 8. MVP 边界

当前版本保持轻量，只做一件事：基于本地证据生成有用的 handoff snapshot。

明确不做：

- Web 应用
- 数据库
- 账号系统
- 实时协作
- 自动提交代码
- 任务管理系统
- 完整语义代码分析

这些能力只有在 handoff snapshot 已经被证明有价值后，才值得继续扩展。

## 9. 500 字以内简版

RelayStack 是一个以 skill 形式落地的 AI 编程工作流增强层，用来解决 AI 编程中的交接断点。当前 AI agent 已经能完成很多开发任务，但上下文常分散在聊天记录、Git diff、任务说明和个人判断里；多人或多 agent 接力时，接手者很难快速知道目标、进度、改动原因、风险和下一步。

RelayStack 会基于本地工作区证据生成 handoff snapshot，记录当前目标、改动范围、已完成和未完成事项、阻塞风险、项目上下文、下一步动作和验证方式；也可以接入本地 AgentRecord，展示多个 agent 的职责、写入范围、采纳状态、冲突和验证结论。

它面向使用 AI 编程工具的开发者、研发团队、项目负责人和 hackathon 团队。作品亮点是轻量、证据优先、交接导向、并行边界可见，并用 handoff success rate 衡量接手质量。第一版不做平台、数据库或任务系统，只专注生成一份能让下一个人或 agent 继续工作的交接快照。
