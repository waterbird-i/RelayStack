# Requirements Attractor

This directory holds settled, implementation-ready requirements.

Keep:

- capability goals
- user-visible behavior
- acceptance criteria
- explicit non-goals
- behavior changes discovered during fixes
- settled product or capability decisions from `rs-decide`

Do not keep:

- raw ideas
- duplicate design details
- temporary debugging hypotheses

Typical writers: `rs-req`, `rs-feat-design`, `rs-feat-accept`,
`rs-issue-fix`, `rs-decide` when a decision changes capability boundaries.
